# Quick Start

The fastest way to start a Context Tree is to let your coding agent do it.

## Paste this into your coding agent

Open Claude Code, Cursor, or any coding agent and paste:

```
I want to set up a Context Tree for my organization.

1. Clone the reference tree:
   git clone https://github.com/agent-team-foundation/first-tree.git
   
2. Read every file in the repo. This is the Context Tree standard.
   Understand the structure: NODE.md, ownership, members, frontmatter.

3. Create a new tree for my organization:
   - Ask me what my organization is called
   - Ask me what domains/areas my team works on
   - Ask me who is on my team (humans and agents)
   - Ask me who owns which domains

4. Based on my answers, create a new repo named after my organization:
   mkdir <org-name>-tree
   cd <org-name>-tree
   git init

5. Generate the full tree structure:
   - Root NODE.md with my org name and description
   - Domain folders organized by concern (not by repo or team)
   - NODE.md in every folder with title and owners
   - A members/ folder with personal nodes for each team member
   - An AGENT.md with before/during/after task instructions

6. Use first-tree as the reference for format, frontmatter,
   and ownership rules. Follow the spec exactly.
```

The agent will clone first-tree, learn the standard, then walk you through setting up your own tree. It will ask you about your org name, your team, your domains, and create everything for you.

This is the intended workflow. The tree is designed to be created and maintained by agents.

## After setup

Once your tree exists:

1. **Push it.** The tree is a git repo. Push to GitHub.
2. **Point your agents at it.** Configure your coding agent to read the tree before every task. The AGENT.md tells them how.
3. **Start recording decisions.** Every design decision, every "why we chose X over Y," goes in a leaf node.
4. **Let agents maintain it.** After every task, the agent asks: does the tree need updating?

## Manual setup

If you prefer to do it by hand, the minimal structure is:

```
my-tree/
├── NODE.md          ← root: what is this org
├── members/
│   └── NODE.md      ← who is on the team
└── AGENT.md         ← instructions for agents
```

Then grow from there. See [03.04 Growing Your Tree](03.04-growing-your-tree.md).
